# Mirror Scope

This page explains what `dmBot` covers, which materials remain Gitee-first, and where to continue when GitHub coverage is not enough.

## Repository Roles

- `dmBot`: English public entry repository with high-frequency English paths.
- `damiao`: Chinese primary repository and the more complete Chinese source.
- Both repositories should keep the same public entry intent, but they do not need identical coverage depth.

## What GitHub Should Still Expose

- root README first-visit guidance and common questions
- major public category entries, especially `1`, `5`, `6`, `8`, `9`, and `B`
- cross-repository links, language switches, and supported continuation pages
- entry links into folder-local workflow docs

## What Still Commonly Remains Gitee-First

- fuller Chinese model materials and history
- some broader tool branches and lower-frequency documents
- fuller Canopen public materials
- deeper `orin-board` content and history

## Current Maintenance Boundaries

- `orin-board` keeps its current directory and documentation pattern.
- `A.Open Source Series` is maintained in its current structure.

## When GitHub Looks Incomplete

1. Go back to root [README.md](../../README.md) and pick the right task path first.
2. Use the category or subtree README to confirm whether the current page is an entry page or a continuation page.
3. If the page already says it is Gitee-first, continue in Gitee instead of guessing deeper GitHub paths.
4. If you came from an old Chinese-path bookmark, open [path-mapping.md](path-mapping.md).

## Direct Advice For Customers

- Need the more complete Chinese source: continue in Gitee `damiao`.
- Need the English entry repository: stay in `dmBot`.
- Supported continuation pages are part of the normal navigation path.
