# Security Policy

This repository is documentation-first.

## Reporting

- Do not post exploit details, credentials, or other sensitive material in a public issue.
- If the issue belongs to a submodule, follow that submodule's security process first.
- If no private reporting channel is available for the affected repository, open a minimal public report and label the affected path.

## Scope

- Broken links and stale docs are not security issues, but they should still be fixed through the normal maintenance flow.
- If a security concern touches published binaries, examples, or scripts, keep the report narrow and factual.
